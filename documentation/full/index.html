How to host a dueling game ?
To start, you must have a working ECP to host a game in Starblast.
If you don’t own one, you won’t be able to host a Starblast modding game.
You may now open modding space tab and remove any code from it like it is shown bellow.
Next you will have copy and paste the Meg's dueling code into the empty modding tab
( You can download the full version at the top )
Once you've done that all that you have to do is to press start and...
Woosh! enjoy the mod!
How to add a custom color to my name ?
There is nothing more simple than that, to do so head at the top of the code.
You should see at line 39/40 something like this:

    // LeaderBoard Feature: (leave thoses variables empty if you don't want a special color on the leaderboard)
    Your_in_game_NAME: "", // Put here your "in Game Name"!   ex: "PMGL"
    Your_in_game_COLOR: "", // Put here your "Favorite Color" using a Hex code (they usally look like this: #6a65ff)!   ex: "#ec371b"
You have a bunch of explanations there. If you understood how they works, good job! You can now play with a custom name. If not, continue reading.
So, first you have:

    Your_in_game_NAME: "", // Put here your "in Game Name"!   ex: "PMGL"
This is the section where you will write your in-game name. It should be written within the quotation marks exactly as it appears in Starblast.
To see an example, here is my name correctly added:

    Your_in_game_NAME: "Megalodon", // Put here your "in Game Name"!   ex: "PMGL"
Then you have:

    Your_in_game_COLOR: "", // Put here your "Favorite Color" using a Hex code (they usally look like this: #6a65ff)!   ex: "#ec371b"
This is the section where you will need to input the color for your name. It should be written within quotation marks, just like your previously added nickname.
To see an example, here is my color correctly added:

    Your_in_game_COLOR: "#005cb9", // Put here your "Favorite Color" using a Hex code (they usally look like this: #6a65ff)!   ex: "#ec371b"
How to Kick a player from the game ?
While your mod is running, simply write the following command in the console:

    > info█
Once you press "ENTER," the mod will reply with something like this:

    > info
      Total amount of aliens: 0
      Total amount of asteroids: 0
      Total amount of players: 2
       
      Banned players: Lotus
       
      Player's and their index's:
        
      index: 0, Name: Megalodon, Type: 613, Alive: true
      Coordinates: X: 7, Y: 3
      index: 1, Name: Kylo, Type: 601, Alive: false
      Coordinates: X: 105, Y: 356
    > █
While looking at it, you can see a few pieces of information.
Firstly, it is obvious that you can see the following:

    Total amount of aliens: 0
    Total amount of asteroids: 0
    Total amount of players: 2
                    
    Banned players: Lotus
This section displays the number of aliens, asteroids, and players in the game. These details can be useful when modding the game.
Directly below this information are the names of banned players.
After this, we have all the interesting stuff.

    Player's and their index

    index: 0, Name: Megalodon, Type: 613, Alive: true  // that is one player
    Coordinates: X: 7, Y: 3                            
    index: 1, Name: Kylo, Type: 601, Alive: false  // that is another player
    Coordinates: X: 105, Y: 356
This information shows the index and name of a player, which can be useful in identifying the player you want to kick. You can also see the ship they have, whether they are dead or not, and their coordinates.
To kick a player, you only need the first line:

    index: 0, Name: Megalodon, Type: 613, Alive: true
Watch carefully for the names, choose your target, and monitor their index. In my case, it's 0. The command you will be using to kick someone is:

    > kick()
    > █
Remember to write down the player's index in parentheses, like this:

    > kick(0)
    > █
Once you press "ENTER", this is what you should see:

    > kick(0)
      Player: Megalodon, index: 0 Has successfully been kicked
 
    > █
This means that you have successfully kicked someone!
You may also add a custom reason, but note that this step is optional. If you choose to set a custom reason, write it as follows:

    > kick(0,"Not skilled enough to survive")█
You can put anything you want between the quotation marks.
warning If an error occurs, please restart the mod.
How to Ban a player from the game ?
Banning a player is also very easy. The ban system is based on the player's name, which means that to circumvent a ban, the player only needs to change their name.
There are no other ways to ban a player. (Please DM me on Discord if you have any other ideas on how we could improve the banning system. Credits will be given.)
To ban a player from the console, as with the "kick" command, you will need to type:

    > info█
(We will skip a few parts. For more details, check: Kick)
o ban a player, simply write their index in the command.

    > ban(index)
    > █
Once you press "ENTER," you should see this:

    > ban(1)
      Player: Gilles, index: 1 Has successfully been banned
      ❗INFO Type: banlist, to see all of the banned players.
    
    > █
Congratulations on successfully banning your first player!
However, you can also add a custom reason with the ban command, just like you can with the kick command.

    > ban(1,"You cannot rejoin now")
      Player: Gilles, index: 1 Has successfully been banned
      ❗INFO Type: banlist, to see all of the banned players.

    > █
How to UnBan a player from the game ?
Unlike the "Kick" command, when a player is "banned," they cannot rejoin using their original name. To allow the player to rejoin, you must use the "Unban" command.
To do so, use the following command:

    > banlist█
Upon pressing the "ENTER" key, a list of all players that have been banned in the current game will appear, along with the reason for their ban. The list is presented in a clear and organized manner, making it easy to find the information you need. Additionally, the ban reason provides important context for understanding the circumstances surrounding the ban.
chevron_rightusage exmple:

    > banlist
      Index: 0, Name: Megalodon, Reason: Disturbing duels
      Index: 1, Name: Nébuleuse, Reason: Being annoying, and Disturbing duels
      ❗INFO Type: unban(index), to unabn a player.
                      
    > █
To unban a previously banned player, locate their index number on the ban list and use it in the unban command. For example, to unban Nébuleuse, find their index number on the list and type "unban(index)" in the command prompt.
This will allow Nébuleuse to rejoin the game.
chevron_rightusage exmple:

    > unban(1)
      Player: Nébuleuse, reason: Being annoying, and Disturbing duels, Has successfully been unbanned
    > █
Well done! You now know how to Ban and Unban someone from Meg's Dueling Starblast Custom Game!
SET Command: Ship Type
The `ship.type` is a way to recognize a ship without his name.
The easiest way to explain it is this:
Let's examine the Fly, which is listed as the first ship and is categorized as a Tier 1 ship. Its corresponding code or identification number is 101.
We can also see with the A-Speedster, The ship is in the fifth position its lane and is classified as a Tier 6 ship. As such, its corresponding code or identification number is 605.
The same logic is applyed to every ships on Starblast.
SET Command: Ship Crystals
You basically don't need to set any value, its automatically giving the right amount of gems.
if you want to change the ship type, stats, and crystals amount at the same time, you can do:

    > set(0, 602, 560)█
Here, 560 represents the amount of gems. This number can be changed as desired within a range of 0 to 999.
SET Command: Ship Stats
Stats are a little harder. You may leave that empty if you don't know how to use it, since in order to use it you must understand the ships basics like the ships classification
First, it's important to understand that all ships work in a way that their level is equivalent to the number of upgrades they can have per category. For example, let's take the X-Warrior. The X-Warrior is a Tier 4 ship, and according to the system, its code/id is 402 because he is the second ship of his list.
Knowing this, you should also be aware that there are 8 categories for the stats:
shieldShield
chevron_rightCapacity
chevron_rightRegen
boltLaser
chevron_rightCapacity
chevron_rightRegen
chevron_rightDamage
chevron_rightSpeed
speedMovement
chevron_rightSpeed
chevron_rightAgility
With that in mind, let's move on to the coding part.
To remove upgrades from a ship, the stats should be set to 0, and would look like this in the console:

    > set(0, 601, 0 ,0)█
However, to fully upgrade a ship, you need to replace the "0" with actual numbers. For instance, consider the following example: 24354321. These numbers represent:
shieldShield
chevron_rightCapacitytrending_flat2 Upgrades
chevron_rightRegentrending_flat4 Upgrades
boltLaser
chevron_rightCapacitytrending_flat3 Upgrades
chevron_rightRegentrending_flat5 Upgrades
chevron_rightDamagetrending_flat4 Upgrades
chevron_rightSpeedtrending_flat3 Upgrades
speedMovement
chevron_rightSpeedtrending_flat2 Upgrades
chevron_rightAgilitytrending_flat1 Upgrades
At the end the command will look like this:

    > set(0, 601, 0, 24354321)
    > █
To fully upgrade your ship, you do not need to input 66666666 as the stats value. since the default value is 88888888.
It would look like this using the default values:

    > set(0, 601)
    > █
